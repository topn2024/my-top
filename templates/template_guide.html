📚 提示词模板使用指南

一、模板概述

提示词模板系统允许您自定义AI分析和文章生成的行为，通过配置不同的参数和提示词，可以针对不同行业、平台和目标受众生成更精准的内容。

🎯 模板的作用

  • 标准化输出：确保生成的内容符合特定行业或平台的要求
  • 提升质量：通过精心设计的提示词提高AI输出的专业度和相关性
  • 批量复用：一次配置，多次使用，提高工作效率
  • A/B测试：创建多个版本的模板，对比效果并优化

📋 模板工作流程

  1. 分析阶段：使用"分析阶段提示词"对公司信息进行深度分析
  2. 生成阶段：基于分析结果，使用"文章生成阶段提示词"创作内容
  3. 发布阶段：将生成的文章发布到指定平台

二、配置参数说明

📝 基础信息参数

参数名称 必填 说明 示例
模板名称 ✅ 是 模板的显示名称，用于识别和选择 科技公司-知乎平台专用
模板代码 ✅ 是 唯一标识符，建议使用英文和下划线 tech_zhihu_v1
描述 ❌ 否 模板用途和特点的详细说明 适合科技类公司在知乎平台的推广

💬 提示词参数

参数名称 阶段 说明
系统提示词 (System Prompt) 分析/生成 定义AI的角色、专业背景和行为准则
用户模板 (User Template) 分析/生成 实际发送给AI的任务描述，支持变量替换

🏷️ 标签参数

参数名称 格式 说明 示例
行业标签 逗号分隔 适用的行业类型，用于智能匹配 tech, ai, cloud
平台标签 逗号分隔 目标发布平台 zhihu, csdn, juejin
关键词 逗号分隔 内容相关的关键词，用于SEO和匹配 AI, 云计算, 创新

⚙️ AI配置参数

参数名称 范围 默认值 说明
Temperature
温度参数		 0.0 - 1.0 0.8 控制输出的创造性和随机性。数值越低输出越保守和确定，数值越高输出越有创意和多样性
Max Tokens
最大令牌数		 500 - 10000 3000 生成内容的最大长度限制（1个token约等于1.5个汉字，3000 tokens约2000字）
💡 Temperature 使用建议：
  • 0.3-0.5：技术文档、分析报告等需要精确性的内容
  • 0.6-0.8：营销文章、产品介绍等平衡创意和准确性
  • 0.8-1.0：创意文案、故事性内容等需要高度创造性

📊 状态参数

状态 说明 使用场景
草稿 (draft) 模板正在编辑中，不会被使用 新建或测试中的模板
活跃 (active) 模板可以被正常使用 已验证效果良好的模板
归档 (archived) 模板已停用但保留记录 过时或不再使用的模板

三、可用变量（插件）

在用户模板中，您可以使用以下变量来动态插入内容。变量使用 {{variable_name}} 格式。

📌 分析阶段可用变量

变量名 说明 示例值
{{company_name}} 公司/产品名称 智云科技有限公司
{{company_desc}} 公司/产品描述 专注于AI云计算解决方案...
{{uploaded_text}} 上传文档的内容 （用户上传的TXT/PDF内容）

📌 文章生成阶段可用变量

变量名 说明 示例值
{{company_name}} 公司/产品名称 智云科技有限公司
{{analysis}} 分析阶段的输出结果 （AI生成的公司分析内容）
{{angle}} 文章切入角度 技术创新、行业应用、用户案例等
📝 使用示例：
请分析以下公司信息： 公司名称：{{company_name}} 公司描述：{{company_desc}} {% if uploaded_text %} 补充资料： {{uploaded_text}} {% endif %} 请从以下角度进行分析： 1. 核心竞争力 2. 目标用户群体 3. 行业地位

四、模板语法

系统使用 Jinja2 模板引擎，支持以下语法：

🔹 变量替换

语法：{{variable_name}} 示例：欢迎 {{company_name}} 使用本平台

🔹 条件判断

语法： {% if condition %} 内容A {% else %} 内容B {% endif %} 示例： {% if uploaded_text %} 根据您提供的文档：{{uploaded_text}} {% else %} 根据您的描述：{{company_desc}} {% endif %}

🔹 循环遍历

语法： {% for item in list %} {{item}} {% endfor %} 示例： 请从以下角度分析： {% for angle in ['技术创新', '市场定位', '用户价值'] %} - {{angle}} {% endfor %}
⚠️ 注意事项：
  • 变量名区分大小写
  • 未定义的变量会被替换为空字符串
  • 避免在系统提示词中使用变量（系统提示词应该是静态的）

五、配置案例

📘 案例1：科技公司 + 知乎平台

模板名称：科技公司-知乎平台专用模板

模板代码：tech_zhihu_v1

行业标签：tech, ai, software

平台标签：zhihu

分析阶段 - 系统提示词：

你是一位资深的科技行业分析师，擅长分析科技公司的技术优势、市场定位和创新能力。你的分析风格专业、客观，注重数据和事实。

分析阶段 - 用户模板：

请深入分析以下科技公司： 【公司名称】{{company_name}} 【公司介绍】 {{company_desc}} {% if uploaded_text %} 【补充资料】 {{uploaded_text}} {% endif %} 请从以下维度进行分析： 1. 核心技术与创新点 2. 目标市场与用户群体 3. 竞争优势与差异化 4. 发展潜力与行业趋势 5. 适合知乎平台的推广角度（3-5个） 请以结构化的方式输出分析结果。

文章生成阶段 - 系统提示词：

你是一位知乎优秀答主，擅长撰写科技领域的深度文章。你的写作风格： 1. 专业但不失通俗，能将复杂概念用简单语言解释 2. 注重逻辑性和结构化，善用小标题和列表 3. 结合实例和数据，增强说服力 4. 符合知乎社区的内容偏好和调性

文章生成阶段 - 用户模板：

基于以下公司分析，撰写一篇适合知乎平台的推广文章： 【公司名称】{{company_name}} 【分析结果】 {{analysis}} 【文章角度】{{angle}} 要求： 1. 标题吸引人但不夸张，符合知乎调性 2. 开篇引入话题，激发读者兴趣 3. 正文结构清晰，使用小标题和列表 4. 自然融入公司/产品介绍，避免硬广 5. 字数2000-3000字 6. 结尾引导互动或思考 请直接输出文章内容（包括标题）。

AI配置：

  • Temperature: 0.7（平衡创意和准确性）
  • Max Tokens: 3500（确保文章长度充足）

文章生成阶段 - 系统提示词：

六、最佳实践

✅ 提示词编写原则

  1. 明确角色定位：在系统提示词中清晰定义AI的角色和专业背景
  2. 具体化要求：避免模糊描述，给出具体的输出格式和要求
  3. 结构化输出：使用编号、小标题等方式组织输出内容
  4. 适度约束：既要给AI发挥空间，又要避免偏离主题
  5. 迭代优化：根据实际效果不断调整和改进

🎯 标签使用建议

  • 行业标签：使用通用的行业分类，如 tech, finance, education, healthcare
  • 平台标签：使用平台英文简称，如 zhihu, csdn, juejin
  • 关键词：包含3-8个核心关键词，兼顾行业术语和通俗表达

⚡ 性能优化建议

  • 合理设置 Max Tokens：根据实际需要设置，避免浪费
  • Temperature 调优：从0.7开始测试，根据效果调整
  • 模板复用：相似场景可以共用一个模板，通过变量实现差异化

🔒 安全性建议

  • 不要在提示词中包含敏感信息（API密钥、密码等）
  • 避免可能导致有害内容生成的指令
  • 定期审查模板使用情况和输出质量
  • 草稿状态测试充分后再改为活跃状态

📊 效果评估方法

  1. 初步测试：创建模板后先设为草稿，用真实数据测试3-5次
  2. 对比分析：与默认模板或其他模板对比生成效果
  3. 数据跟踪：查看模板的使用次数和成功率统计
  4. 用户反馈：收集实际发布后的阅读量、互动数据
  5. 持续优化：根据数据反馈调整提示词和参数

七、常见问题（FAQ）

❓ Q1: 模板代码重复了怎么办？

A: 模板代码必须唯一。如果提示重复，请修改代码，建议在原有基础上添加版本号，如 tech_zhihu_v2。

❓ Q2: 变量没有被正确替换？

A: 检查以下几点：

  • 变量名拼写是否正确（区分大小写）
  • 是否使用了正确的语法 {{variable_name}}
  • 该变量在当前阶段是否可用（参考"可用变量"章节）

❓ Q3: 生成的内容质量不理想？

A: 可以尝试：

  • 优化系统提示词，更明确地定义角色和风格
  • 在用户模板中添加更具体的要求和示例
  • 调整 Temperature 参数（降低获得更稳定输出，提高获得更多创意）
  • 增加 Max Tokens 以允许更长的输出
  • 提供更详细的公司描述或上传补充文档

❓ Q4: 如何测试模板效果？

A:

  1. 创建模板后先设置为"草稿"状态
  2. 在推广平台页面手动选择该模板
  3. 输入测试数据，查看生成效果
  4. 满意后再将模板状态改为"活跃"

❓ Q5: 一个模板可以用于多个平台吗？

A: 可以。在平台标签中填入多个平台即可，用逗号分隔，如 zhihu, csdn, juejin。但建议针对不同平台的特点定制专门的模板以获得更好效果。

❓ Q6: 如何查看模板的使用统计？

A: 在模板管理页面，每个模板卡片上会显示使用次数。点击"查看日志"可以看到详细的使用记录和审计日志。

❓ Q7: 可以导入/导出模板吗？

A: 目前可以通过API进行导入导出。未来版本将提供可视化的导入导出功能。

❓ Q8: Temperature 和 Max Tokens 设置多少合适？

A: 推荐值：

  • Temperature: 0.7-0.8（营销文章）、0.5-0.6（技术文档）
  • Max Tokens: 3000-3500（长文章）、2000-2500（中等长度）、1500-2000（短文章）

❓ Q9: 修改模板会影响已生成的内容吗？

A: 不会。已经生成的内容不受模板修改影响。模板修改只影响后续的新生成任务。

❓ Q10: 如何删除模板？

A: 点击模板卡片上的"删除"按钮。建议：如果模板可能还有用，可以先设置为"归档"状态而不是直接删除。
